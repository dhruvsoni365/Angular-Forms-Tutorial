üìã Issue: Form validation not working in Angular 12 due to strict null checking

Problem Statement

Angular 12+ introduced stricter TypeScript null checking. When accessing form controls using FormGroup.get(), TypeScript now requires handling the possibility that the method might return null. Without optional chaining, this causes compilation errors.

Solution Applied

Added optional chaining operator (?) to all form control property accesses in /vercel/sandbox/reactive-forms/src/app/app.component.html

Code Examples

‚ùå Before (Angular 6 - Causes errors in Angular 12+)

<input type="text" [class.is-invalid]="userName.invalid && userName.touched" formControlName="userName" class="form-control"> <div *ngIf="userName.invalid && userName.touched"> <small *ngIf="userName.errors?.required"> Username is required </small> </div>

‚úÖ After (Angular 12+ Compatible)

<input type="text" [class.is-invalid]="userName?.invalid && userName?.touched" formControlName="userName" class="form-control"> <div *ngIf="userName?.invalid && userName?.touched"> <small *ngIf="userName?.errors?.required"> Username is required </small> </div>

Why This Fix Works

  • Optional Chaining: The ? operator safely handles null/undefined values
  • TypeScript Compliance: Satisfies Angular 12+ strict null checking requirements
  • Runtime Safety: Prevents runtime errors if form controls are not initialized
  • Backward Compatible: Works with both older and newer Angular versions
